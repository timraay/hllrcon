Thank you for considering contributing to **hllrcon**!

Pull requests (PRs) are welcome. To keep collaboration smooth and effective, please consider the following:

### Description
- Explain what changed: In your PR description, clearly summarise the change(s) you made.
- Explain why it changed: Describe the motivation.

### Tests and linting
- This project uses `pytest` for testing, `mypy` for type checking and `ruff` for linting.
- Add or update tests when you add or change behavior.
- Run the project's linters/formatters before opening a PR.
- Pull requests must have no type or linting errors. Unit tests may not fail and must achieve 100% line coverage.

Contributions keep this project healthy and improving. I look forward to your PRs!
